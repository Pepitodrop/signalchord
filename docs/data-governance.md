# Data governance

## Source classification

Each source record contains owner/publisher, acquisition method, terms/robots review state, license category, allowed content fields, raw retention, derived-data permission, geographic restrictions, crawl rate, deletion contact and review date. A collector cannot run without an active ingestion policy.

## Provenance classes

- **Source report:** what a source explicitly published.
- **Model extraction:** structured interpretation generated by a named model/version.
- **Graph inference:** calculation based on graph topology or multiple extractions.
- **Human verification:** an authenticated reviewer decision with rationale.

The UI and APIs retain these labels and never collapse them into one generic fact state.

## PII classification

| Class | Example | Handling |
|---|---|---|
| P0 | system/public non-personal metadata | standard controls |
| P1 | public professional person data | purpose limitation, provenance and retention |
| P2 | user/account/contact data | encryption, strict tenant access, export/deletion |
| P3 | sensitive allegations or inferred personal attributes | review gate, restricted access, minimized retention |
| P4 | secrets/authentication material | secret manager only; never events or logs |

## Retention

Retention is the minimum of source rights, tenant plan and legal requirement. Raw documents use immutable object versions plus deletion tombstones; event retention is topic-specific; derived graph elements can be retracted/closed temporally while preserving a minimal audit record where legally permissible.

## Deletion workflow

1. Disable the source or identify subject/user scope.
2. Publish a signed deletion request and tombstone.
3. Delete or quarantine raw objects and search projections.
4. Retract/delete eligible graph facts and evidence links.
5. Expire caches and downstream exports where controlled.
6. Record completion, exceptions and legal holds in audit storage.
7. Reconciliation confirms no active projection remains.

## User rights

Provide tenant data export, personal account export, account deletion, token revocation and notification preference controls. Exports contain provenance and status so disputed/retracted material is not misrepresented.

## Quality and review

Entity-resolution precision, claim precision, confidence calibration, contradiction review precision, duplicate reduction and source diversity are measured by language/source segment. Model upgrades require regression fixtures and a rollback path.

## Ownership

Product owns purpose and customer controls; Data Governance owns source policy and retention; Security owns access/encryption/audit; ML owns extraction quality; Platform owns lifecycle execution; Legal reviews licensing, copyright and sensitive-claim processes.
